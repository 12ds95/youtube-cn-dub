# Phase 2 全文翻译质量优化 — ROADMAP

## 问题背景

本项目的中文配音管线面临两个核心难题：

### 难题 1: 翻译质量
分段翻译 + LLM 行为不确定性导致：
- **跨段内容污染**：LLM 将相邻段内容混入当前段（已有 neighbor dedup 缓解）
- **上下文丢失**：批次边界处的段缺乏完整上下文
- **幻觉/漏译**：LLM 对短碎片段（Whisper 断句产生）翻译不稳定
- **等时压缩加剧问题**：refine 迭代为适配 TTS 时长反复改写译文，进一步偏离原意

### 难题 2: TTS 语音时长
中文译文必须适配每段英文的时间窗口（目标 CPS 3.5-6.0）。
Phase 1 管线已通过 isometric 多候选 + jieba 时长估算 + refine 迭代基本解决此问题。

### 核心洞察
Phase 1 解决了 TTS 时长问题，副产品是**每段中文字数的 ground truth**（即多少汉字能在该时间窗口内自然朗读）。Phase 2 可以以此为约束，专注优化翻译质量。

## Phase 2 方案：全文翻译 + DP 切分

### 两阶段架构
1. **Phase 1**（现有管线）：分段翻译 → isometric → refine → TTS → 产出每段字数 budget
2. **Phase 2**（新脚本 `phase2_translate.py`）：全文连贯翻译 → 多候选 → DP 按 budget 切分 → 选最优

### 当前实现（v0 基线）

已实现的 `phase2_translate.py` 流程：
1. 从 `segments_cache.json` 提取每段汉字数作为 budget
2. 拼接所有 `text_en` 为完整英文
3. 生成 5 个全文翻译候选（不同 temperature/风格）
4. DP 算法按 budget 在句子/子句边界最优切分
5. 评分选优，输出 `segments_cache_phase2.json`

### v0 实验结果（测试视频 zjMuIxRvygQ, 73 段）

**DP 切分精度**：极高
- MAE = 0.6 字/段，73 段 100% 在 ±2 字内
- 候选 1 (T=0.3) 得分最优 (total_score=2.7)

**翻译质量**：全文翻译明显更连贯自然

**关键问题：语义错位**
纯按字数切分无法保证段内容与英文原文语义对齐。全文翻译的语序和详略分布不同于分段翻译，导致切分后段内容错位。例如：
- 段 10 英文讲四元数，但 P2 切分到的中文是"值得亲自体验"
- 段 5 英文讲合作不同，P2 切分到的是"强烈推荐你去关注"

## 待探索的改进方向

### 方向 1: Prompt 段边界标记
在翻译 prompt 中用 `|||` 等标记标注英文段边界，让 LLM 在翻译时保持段序对应：
```
请翻译以下英文，用 ||| 标记对应原文的分段边界：
In a moment I'll point... ||| calling Explorable Videos... ||| you may know is...
```
- 优势：简单，LLM 可能自然保持对齐
- 风险：分段标记可能影响翻译流畅度，回到"分段翻译"的老路
- 需实验：标记密度（每段 vs 每 5 段）对翻译质量的影响

### 方向 2: 英中 Sentence Alignment
全文翻译后，用英文原文分段作为锚点，做英中句对齐：
- 用 sentence embedding（如 text2vec、labse）计算英中段相似度
- 基于相似度矩阵做 DTW/DP 对齐，找到最优英中段映射
- 优势：不侵入翻译过程，翻译自由度最高
- 风险：中文全文翻译可能合并/拆分原文信息，导致对齐模糊
- 依赖：需要额外的 embedding 模型

### 方向 3: 两步法（翻译 + LLM 分句）
1. LLM 输出全文连贯翻译
2. 第二个 LLM 调用：给出英文分段 + 中文全文 + 每段 budget，让 LLM 按语义切分中文
- 优势：利用 LLM 的语义理解能力做对齐
- 风险：额外 API 成本，LLM 切分可能不精确
- 可与 DP 混合：LLM 粗切 → DP 精调

### 方向 4: 混合策略
- 全文翻译时按 5-10 段为组加 `|||` 标记（粗粒度对齐）
- 组内用 DP 精确切分（细粒度匹配 budget）
- 平衡翻译自由度和段对齐

## 数据需求

上述方向的选择需要更多数据支撑：
- 不同视频类型（教学、叙事、对话）的翻译错位程度
- 不同 LLM 模型对段标记的遵从度
- Sentence alignment 方法在中英视频旁白上的准确率
- 各方向的 API 成本对比

**计划**：等积累 5+ 视频的 Phase 1/Phase 2 数据后，再做系统评估选择最优方向。
