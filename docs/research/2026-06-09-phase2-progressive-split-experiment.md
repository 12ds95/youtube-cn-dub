# Phase 2 渐进分段翻译实验报告

## 实验目标

在 Phase A (Self-Refine 迭代翻译) 发现全文翻译存在全局语序偏移的基础上，验证**渐进分段翻译 (Progressive Split Translation)** 策略：将全文分为 k 组分别翻译，搜索「翻译流畅度 vs 对齐精度」的最优分段粒度。

## 方法

### 核心架构

1. **渐进分组**: 遍历 group_schedule `[1, 2, 3, 4, 5, 6, 8, 12, 16, 24]`，每个粒度独立评估
2. **分组翻译**: 每组提供前后邻组上下文 (英文 + 中文结尾)，组内自由翻译
3. **组内 ||| 标记**: 大组 (>5 段) 每 5 段插入 ||| 分段标记，帮助保持语序
4. **DP 切分**: 组翻译拼接后，用 DP 按 budget 最优切分为 N 段
5. **三维评估**:
   - LLM 自评 (忠实度/流畅度/完整性/配音适合度, 各 1-10) — 权重 40%
   - Sentence embedding 对齐 (paraphrase-multilingual-MiniLM-L12-v2 cosine sim) — 权重 40%
   - Budget 偏差 (MAE) — 权重 20%
6. **穷举搜索**: 不做早停，遍历全部分组级别，取全局最优

### 关键设计决策

- **上下文传递**: 小组 (≤10段) 传 2 句上下文，大组传 1 句
- **字数约束**: 每组给出 85%-115% 的严格字数范围提示
- **统一 Prompt**: 所有组大小使用相同的 TRANSLATE_SYSTEM_PROMPT (实验证明专用严格 prompt 有害)
- **多候选**: 每级生成 2-3 个温度候选 (T=0.3, 0.5, 0.7)，取最优

### 测试数据

- 视频: `zjMuIxRvygQ` (73 段, 英文 3890 字符, Budget 总计 1454 字)
- 模型: qwen3-coder-next (DashScope API)
- Embedding: paraphrase-multilingual-MiniLM-L12-v2 (离线加载)

### 逐迭代详细分析

**迭代 1 — 基线 (2^k schedule)**
- Schedule: [1, 2, 4, 8, 16, 32, 64]
- 发现: 4 组最优 (combined 74.8), 1 组对齐差 (0.371), 8+ 组 LLM 分崩溃
- 关键洞察: 存在倒 U 型曲线 — 组数过少则对齐差, 过多则幻觉严重

**迭代 2 — 反幻觉 prompt**
- 新增 TRANSLATE_SMALL_GROUP_PROMPT: "不要补充背景知识、不要添加例子、不要发挥"
- 结果: 2-3 组质量反而下降 (combined 73.8)
- 分析: 过度限制导致翻译僵硬, 自然度受损
- 操作: 撤回, 统一使用 TRANSLATE_SYSTEM_PROMPT

**迭代 3 — 细粒度 schedule**
- Schedule: [1, 2, 3, 4, 6, 8, 12, 16, 24] (在 2-4 之间加密)
- 结果: 3 组最优 (combined 75.8, align 0.441)
- 发现: 3 组比 4 组略优, 证实最优区间在 2-5 组

**迭代 4 — 多候选 + 聚焦搜索**
- Schedule: [2, 3, 4, 5], 3 候选/级
- 结果: 2 组最优 (combined 73.5) — 但低于 iter 3 的 75.8
- 关键发现: **运行间方差很大**, 同配置不同运行结果可差 2-3 分

**迭代 5 — 穷举搜索**
- 移除连续 2 次变差的早停机制, 遍历全部级别
- 结果: 5 组最优 (combined 75.3, align 0.473)
- 发现: 完整景观确认 6+ 组为断崖 (LLM 评分从 33 骤降至 28), 8+ 组出现幻觉/重复

**迭代 6 — 组内 ||| 标记**
- 大组 (>5 段) 每 5 段插入 ||| 标记, 要求 LLM 在译文对应位置保留
- 结果: 5 组对齐达 0.473 (与 iter 5 持平), 2 组 combined 74.2
- 发现: 标记对大组对齐有帮助, 但不改变整体最优区间

**迭代 7 — 评分权重调整**
- 权重从 LLM 50%/Align 30%/Budget 20% → LLM 40%/Align 40%/Budget 20%
- 结果: 5 组最优 (combined 70.4), **2 组对齐达 0.508** (历史最高)
- 发现: 提高对齐权重后, 2 组方案更突出 (align 0.508, 43/73 段 >0.5 sim)

## 关键发现

### 1. 最优分段区间: 2-5 组 (15-37 段/组)

全部 7 轮实验一致表明:
- **2-5 组**是翻译质量的最优区间
- 每组约 15-37 段 (英文 800-2000 字符) 时 LLM 既能保持全局连贯又不会失控
- 1 组 (全文) 对齐差 (0.37), 6+ 组幻觉/重复严重 (LLM 跌至 28/40)

### 2. 6+ 组存在幻觉断崖

当每组 <12 段时, LLM 翻译质量急剧下降:
- LLM 自评从 33-35/40 骤降至 24-28/40
- 出现信息重复 ("安迪·马塔兹恰克段落重复")
- 出现幻觉添加 (原文未提及的人名、公司名、数据)
- 根本原因: 短片段缺少足够上下文, LLM 倾向用背景知识填充

### 3. 运行间高方差

同一配置不同运行的 combined 分差可达 2-5 分:
- iter 3: 3 组 = 75.8
- iter 4: 3 组 = 73.5 (同配置低 2.3 分)

这意味着单次运行的精确排名不够可靠, 多次运行取均值更稳定。

### 4. 组内 ||| 标记对大组有效

在 >5 段的组内每 5 段插入 ||| 标记:
- 帮助 LLM 保持段间语序
- 5 组的对齐从基线 ~0.45 稳定在 0.47+
- 但对整体 combined 改善有限 (<1 分)

### 5. LLM 自评分辨率有限

- 所有 2-5 组的 LLM 自评集中在 32-35/40
- 无法有效区分对齐好 (0.508) 和对齐中等 (0.44) 的译文
- Embedding 对齐是更可靠的质量区分指标

### 6. 对比 Phase A (Self-Refine)

| 指标 | Phase A 最优 | Phase B 最优 |
|------|-------------|-------------|
| 方法 | 全文 + ||| + Self-Refine | 渐进分段 (2-5 组) |
| Combined | 76.6 | 75.8 |
| Align | 0.466 | 0.508 |
| MAE | 0.6 | 0.59 |
| LLM 自评 | 35/40 | 34/40 |

两种方法各有优势:
- Phase A 的综合分略高 (76.6 vs 75.8), 但依赖 Self-Refine 机制 (实际几乎无效)
- Phase B 的对齐更优 (0.508 vs 0.466, +9%), 且方法更简单直接
- Phase B 的核心优势是 **2 组分段 + 上下文传递** 即可大幅改善对齐

## 最终推荐配置

```python
# phase2_iterative_split.py 推荐参数
group_schedule = [2, 3, 4, 5]   # 聚焦最优区间
n_candidates = 2                 # 每级 2 候选 (T=0.3, 0.5)
context_sentences = 2            # 小组 2 句上下文
intra_markers = True             # 大组 (>5段) 用 ||| 标记
scoring_weights = "40/40/20"     # LLM / Align / Budget
embedding = "paraphrase-multilingual-MiniLM-L12-v2"

# 实际使用时建议:
# 1. 运行 2-5 组全部级别 + 穷举搜索 (不早停)
# 2. 取全局 combined 最优
# 3. 如需更高对齐, 优先选 2 组结果
```

## 后续方向

1. **融合 Phase A + B**: 先分组翻译 (Phase B) 获得高对齐基线, 再做全文 Self-Refine 微调
2. **多视频验证**: 当前仅在 zjMuIxRvygQ (73段) 测试, 需扩展到不同长度/领域视频
3. **自适应分组**: 根据视频长度自动选择最优组数 (如 N/15~N/37)
4. **降低方差**: 每级 3-5 候选取均值, 或固定随机种子确保可复现
5. **更好的对齐评估**: 用 LaBSE 或更大的多语言模型替代 MiniLM, 可能进一步提升对齐区分度
